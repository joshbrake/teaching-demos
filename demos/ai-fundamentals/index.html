What Is a Model?

Analogy
A model is like an experienced home appraiser. Show them a house — its square footage, number of bedrooms, neighborhood — and they'll predict a price. They learned by studying thousands of past sales. A machine learning model works the same way: it finds patterns in data to make predictions about new inputs.

At its core, a model is a function that maps inputs to outputs. You give it data (like house size), and it gives you a prediction (like price).

The model learns this mapping by being shown many examples. A simple model (like a straight line) captures broad trends. A complex model (like a wiggly curve) can capture more detail — but it risks overfitting, memorizing noise instead of learning the real pattern.

Try it: Click on the plot to add data points. Toggle between fit types to see how complexity changes the curve. Watch what happens when you crank up the degree slider.

Key Takeaway
All of AI comes down to this: learning a function from data. The art is choosing the right model complexity — simple enough to generalize, complex enough to be useful.

Tokens & Text

Analogy
You read words. LLMs read tokens — fragments of words, like puzzle pieces. The word "unbelievable" might become three tokens: "un", "believ", "able". Common words stay whole; rare words get split. It's like Morse code: common letters get short signals (E = dot), rare ones get long.

Before a language model can process text, it must convert it into tokens — numerical IDs from a fixed vocabulary.

This process, called tokenization, splits text into subword pieces. Common words like "the" are a single token. Rare or long words get broken into smaller chunks. Even punctuation and spaces become tokens.

Try it: Type anything in the input box and watch it get tokenized in real time. Try names, code, or long words to see how they split.

Key Takeaway
LLMs don't think in words — they think in tokens. This is why they sometimes struggle with spelling, counting letters, or handling rare words: the boundaries don't align with how we think about text.

Next-Token Prediction

Analogy
You know how your phone suggests the next word as you type? An LLM does the same thing — but after reading essentially the entire internet. For every position, it builds a probability distribution over all possible next tokens. It's the world's most sophisticated autocomplete.

The core operation of a language model is simple: given a sequence of tokens, predict what comes next.

The model doesn't just pick one answer — it assigns a probability to every possible next token. "The cat sat on the..." might give high probability to "mat", "floor", "couch", and lower probability to "elephant" or "spacecraft".

Try it: Click on a probability bar to choose that token. It gets appended, and new probabilities appear. Use "Auto" to let the model pick the most likely token automatically.

Key Takeaway
Generation is just repeated prediction. The model predicts one token, appends it, then predicts the next. The entire output is built one piece at a time — the model never "sees" the full response in advance.

Temperature & Sampling

Analogy
Temperature controls how adventurous the model is. Low temperature (0.0) is like always ordering your favorite dish — safe, predictable. High temperature (1.5) is like telling the chef to surprise you — you might get something amazing or something weird. Temperature 1.0 is the model's natural confidence.

The raw model produces logits (scores) for each token. Temperature rescales these before converting to probabilities:

p(token) = exp(logit / T) / sum(exp(logits / T))

Low T sharpens the distribution (one token dominates). High T flattens it (more tokens become likely). This single parameter controls the creativity-reliability tradeoff.

Try it: Drag the temperature slider and watch the bars reshape. Then click "Sample 100" to see which tokens actually get picked at different temperatures.

Key Takeaway
Temperature doesn't change what the model "knows" — it changes how it chooses. T=0 always picks the most likely token. Higher T lets less-likely tokens through, producing more diverse (but potentially less coherent) output.

Attention

Analogy
When you read "The cat sat on the mat because it was tired," you instantly know "it" refers to "the cat" — not "the mat." Your brain looks back and connects the pronoun to the right noun. Attention is the mechanism that lets a model do the same thing: for every token, it decides which other tokens matter most.

Attention is the key innovation in the Transformer architecture (the "T" in GPT). Each token computes a weighted average over all other tokens, where the weights reflect relevance.

This is how models handle long-range dependencies — a token near the end of a paragraph can "look back" at a token from the beginning if it's relevant.

Try it: Click on any token to see which other tokens it attends to. Thicker, brighter arcs mean stronger attention. Try different sentences to see different patterns.

Key Takeaway
Attention lets models figure out context dynamically. Instead of processing text in a fixed window, each token can selectively focus on whichever other tokens are most relevant — no matter how far away they are.

From Chat to Agents

Analogy
A chatbot is like a friend who answers questions. An agent is like a personal assistant who solves problems. Ask a chatbot "What's the weather?" and it gives information. Ask an agent "Should I bring an umbrella?" and it checks the weather, looks at your calendar, sees you have an outdoor meeting at 2pm, and says "Yes, bring one."

The key shift from chatbot to agent is the loop. A chatbot takes one input and produces one output. An agent runs a cycle:

Observe → gather information
Think → reason about what to do next
Act → take an action (call a tool, send a message)
… then observe the result and repeat.

Try it: Click "Step" to walk through each phase of the agent loop, or "Auto-play" to watch it unfold. The transcript below shows the agent's thinking in real time.

Key Takeaway
An agent is a model in a loop. The model itself doesn't change — it's still doing next-token prediction. But by wrapping it in an observe-think-act cycle with access to tools, it can solve multi-step problems autonomously.

Tool Use

Analogy
An LLM by itself is like a brilliant thinker locked in a room with no phone, no computer, no calculator. It can reason impressively, but it can't check facts or take real actions. Tools give the agent hands: a calculator for math, a search engine for current info, a code runner for computation. The agent decides which tool to use, calls it, and incorporates the result.

Tool use is what makes agents practical. The model generates a structured tool call (like a function call), the system executes it, and the result gets fed back to the model.

This is powerful because the model can reason about when to use a tool, which tool to use, and how to interpret the result — combining the model's reasoning with the tool's capabilities.

Try it: Click "Next Task" to see the agent solve different problems by choosing the right tool. Watch how it reasons about which tool is appropriate.

Key Takeaway
Tools extend what AI can do from "thinking" to "doing." A model with tools can browse the web, write and run code, query databases, send messages — anything you can define as a function. This is the foundation of agentic AI.
